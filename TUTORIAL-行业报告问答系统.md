# 行业报告问答系统教程（OpenClaw 实战版）

> 目标：把你持续收到的 PDF 行业报告，沉淀成可检索、可追溯、可问答的知识系统。  
> 本教程基于目录：`~/Git/美妆行业报告`

---

## 1. 系统目标与能力

我们希望实现：

1. **自动入库**：PDF → Markdown（可读）
2. **文本检索**：基于 BM25 的稳定召回
3. **图表增强**：用多模态提取图表关键数字
4. **联合问答**：文本证据 + 图表证据一起回答

最终效果：你问“2025年线上美妆大盘如何”，系统能返回结论，并附带页码级数据证据。

---

## 2. 目录结构

```bash
~/Git/美妆行业报告
├─ raw/                        # 新 PDF 投递目录
├─ knowledge/                  # 结构化 markdown 输出
├─ index/
│  ├─ chunks.jsonl             # 文本分块索引
│  └─ chart_cards.jsonl        # 图表数值卡片索引
├─ assets/                     # 图表增强过程产物（页面图等）
└─ scripts/
   ├─ ingest_reports.py        # PDF 入库（文本）
   ├─ query_reports.py         # 联合检索（文本+图表）
   ├─ render_pdf_pages.py      # PDF 渲染为页面图
   └─ select_chart_pages.py    # 自动筛图表页
```

---

## 3. 第一步：文本入库（基础版）

### 3.1 执行入库

```bash
cd ~/Git/美妆行业报告
python3 scripts/ingest_reports.py
```

### 3.2 入库结果

- 在 `knowledge/` 生成每份报告的 `.md`
- 在 `index/chunks.jsonl` 生成文本分块索引

用途：支持“按关键词/语义相近文本”做证据检索。

---

## 4. 第二步：图表增强（多模态）

仅 `pdftotext` 会丢图表信息，因此加入图表增强链路。

### 4.1 渲染 PDF 页面图

```bash
cd ~/Git/美妆行业报告
python3 scripts/render_pdf_pages.py "2025年度UCO美妆行业报告.pdf"
```

### 4.2 自动筛选图表页

```bash
python3 scripts/select_chart_pages.py "2025年度UCO美妆行业报告.pdf" --topk 12
```

输出：
- 页面图片：`assets/<报告名>/pages/page-xx.png`
- 图表页清单：`assets/<报告名>/chart_pages.json`

### 4.3 多模态抽取后落库

图表抽取结果写入：
- `index/chart_cards.jsonl`（结构化，机器可检索）
- `knowledge/*.chart-cards.md`（可读摘要）

---

## 5. 第三步：联合检索问答

### 5.1 查询命令

```bash
cd ~/Git/美妆行业报告
python3 scripts/query_reports.py "2025年线上美妆大盘表现如何" --topk 6 --chart-topk 6
```

参数说明：
- `--topk`：文本索引返回条数（`chunks.jsonl`）
- `--chart-topk`：图表索引返回条数（`chart_cards.jsonl`）

### 5.2 输出结构

- 文本证据（chunk 片段）
- 图表证据（页码、指标、数值、置信度）

## 7. 日常操作 SOP

### 每次新增报告

1) 把 PDF 放到 `raw/`

2) 运行文本入库

```bash
python3 scripts/ingest_reports.py
```

3) 运行图表增强（建议）

```bash
python3 scripts/render_pdf_pages.py "<报告文件名>.pdf"
python3 scripts/select_chart_pages.py "<报告文件名>.pdf" --topk 12
# 然后执行多模态图表抽取并落库（当前流程已接入）
```

4) 进行联合检索问答

```bash
python3 scripts/query_reports.py "你的问题" --topk 6 --chart-topk 6
```

## 8. 质量控制建议

1. **数字必须可追溯到页码**：回答中保留 page 引用
2. **重要数字二次核验**：高价值结论建议回原图复核
3. **区分“摘要级”与“正文级”**：避免过度推断
4. **宁缺毋滥**：未确认内容不写确定性结论

## 10. Prompt 实战复盘（修正版）：聚焦“行业报告问答系统”

你说得对：这一章应该聚焦你下达“行业报告问答系统”任务时的 prompt，而不是后续 AI 新闻页的 prompt。以下是修正后的复盘。

### 10.1 本项目关键指令链（按时间）

#### 指令 1：先定义需求形态（知识沉淀 + 可检索问答）
**你原始意图（摘录归纳）：**
> 我会定期给你 PDF 行业报告，你要分析消化后用 md 或向量数据库保存到固定位置；后续我提问时你去那里取数并回答。

这个指令一次性给了 4 个关键约束：
1) 数据源：定期 PDF
2) 处理：分析消化，不是只存原文
3) 存储：md/向量（可检索）
4) 目标：后续问答可复用

#### 指令 2：立即落地执行
**你原始指令：**
> 好的，现在就帮我搭建一个

这是典型“从讨论转执行”的触发句，避免无限方案讨论。

#### 指令 3：固定落地路径
**你原始指令：**
> ~/Git/美妆行业报告

路径一旦明确，后续自动化脚本、索引和查询命令都有了稳定锚点。

#### 指令 4：用真实业务问题做验收
**你原始指令：**
> 我们测试一下。我想了解2025年各家抖音直播的情况

这一步非常关键：不是“脚本跑通就算完成”，而是用业务问题反向验收系统质量。

#### 指令 5：提高输出标准（从结论到数字）
**你原始指令：**
> 好的，我想看一些具体的数字

这会强迫系统从泛化总结升级到“数值证据型输出”。

#### 指令 6：追问数据完整性（图表是否丢失）
**你原始指令：**
> 那么你会丢失图表中的信息吗

这一步直接触发“文本检索版 → 图表增强版”的架构升级。

#### 指令 7：确认升级路线
**你原始指令：**
> 是的，需要升级到图表增强版
> 同意

简短但明确，能快速推进到实现阶段。

#### 指令 8：要求教程化沉淀
**你原始指令：**
> 帮我写到 “~/Git/美妆行业报告/TUTORIAL-行业报告问答系统.md”

这让项目从“单次交付”升级为“可复制方法论”。

---

### 10.2 这组 prompt 为什么有效

1. **先定义系统目标，再谈技术手段**
- 你先说“长期可问答”，再说 md/向量；所以方案不会跑偏成一次性摘要。

2. **每一步都有“下一步动作”**
- 搭建 → 指定路径 → 测试问题 → 提高标准 → 升级能力。

3. **验收标准由业务问题驱动**
- “抖音直播表现”“具体数字”比“脚本成功”更能检验系统价值。

4. **持续逼近证据链**
- 从“结论”推进到“数字”，再推进到“图表信息不丢失”。

### 10.3 可复用模板（行业报告问答系统专用）

#### 模板 A：系统搭建指令

```text
我会定期给你【行业】PDF报告。
请你搭建一个可持续问答系统：
1) 入库并结构化（不是只存原文）
2) 存到固定路径【PATH】
3) 至少支持文本检索
4) 最终支持我后续按问题直接取数回答

先给我可运行的 v1，然后我用真实问题验收。
```

#### 模板 B：验收与提标指令

```text
我们测试一下：
【业务问题】

要求：
- 给结论
- 给具体数字
- 给证据来源（页码/图表）
```

#### 模板 C：能力升级指令（图表增强）

```text
当前结果不够，我要图表级数据。
请升级为图表增强版：
- 保留图表关键数值
- 输出结构化卡片
- 查询时支持文本+图表联合检索
```
